Make a Hexo Podcast with GitHub on Windows

今天我来教你如何用 Hexo 在GitHub上托管播客吧

首先下载两个需要的工具软件。

  • Node.js 是运行 Hexo 所需要的环境。下载 Node.js 并安装：
    https://nodejs.org/en/download/
  • Git 是代码版本管理工具，在这个项目中用来向GitHu部署代码。下载 Git 并安装 https://git-scm.com/downloads
  1. 站点的搭建

  • 创建一个GitHub帐号 https://github.com/ 接下来以用户名为 ddfcast 举例

    • Setup 页面可以什么都不填 CompleteSetup 之后直接验证邮箱
    • 登陆之后，点击页面右上角的加号，选择New repository
      ddfcast.github.io ，之后点下面的 Create Repository
    • 访问 https://github.com/settings/ssh/new
    • 保持页面打开，回到桌面

  • 新建一个文件夹用来存放整个站点的文件。这里以文件夹名为 DDFPod 举例

    • 在 DDFPod 上右键，选择 Git Bash Here 以进入 Git Bash 控制台
    • 输入 npm install -g hexo-cli 并 Enter 以安装 Hexo ，Hexo 是静态站点生成器用以从 Markdown 文件生成 HTML 页面和 RSS Feed ，大约需要花费时间不超过10秒
    • 输入 hexo init 并 Enter 以将文件夹初始化为 Hexo 站点
    • 此时再输入 hexo server 或者 hexo s 并 Enter 之后访问 http://localhost:4000 已经可以在本地服务器预览站点，
      在命令行按 Ctrl + C 可以关闭服务器来进行下一步操作

  • 生成 SSH Key

    • 在 GitBash 里
    • 输入 ssh-keygene ，一路回车生成 SSH Key
    • 输入 cat ~/.ssh/id_rsa.pub 并 Enter ，将显示的以 ssh-rsa 开头的长串结果复制（Git Bash里 Ctrl + C 无法复制，建议使用鼠标右键）到剪贴板，并贴入刚才打开的 https://github.com/settings/ssh/new 页面，的 Key 当中，至于 Title ，起一个方便管理的名称，比如你的计算机的名字即可,然后点 Add SSH Key 提交
    • 如果需要更换电脑或者 SSH Key 泄露，建议生成新的 SSH Key ，添加到 GitHub 账号，并删除原来的

  • 输入 npm install hexo-deployer-git --save 并 Enter 以安装 Hexo 的 GitHub 部署插件

  • 从资源管理器进入文件夹 DDFPod，修改文件 _config.yml 结尾部分为如下，注意替换用户名和项目名为你自己的

  1. Podcast的站点配置
  • 输入 npm install hexo-generator-podcast --save 并 Enter 以安装 Podcast 生成器
  • 用文本编辑器（推荐使用 Visual Studio Code）编辑 DDFPod 里的 _config.yml
  • 修改大约第5 ~ 12行的站点（播客）基本信息
  • 修改大约第16行的 url ，例：https://ddfcast.github.io
  • 添加下列内容到配置文件末尾，在 deploy 段落之后
  • 参数说明：
  • 其中 default_thumb是播客logo，路径可以是 URL ，或者在 DDFPod 的 source 文件夹内新建 img 文件夹并放入logo.jpg
    • type 不需要修改，
    • path 是生成的 RSS Feed 位置，按需修改，例子默认为 https://ddfcast.github.io/podcast.xml ，比如需要在 https://ddfcast.github.io/feed/rss.xml 的话，就修改为 /feed/rss.xml
    • limit 限制播客 Episode 数，不过我想不会有人会用到这个的
    • URL 播客主页URL，会显示在 RSS Feed 里，可能会影响到播客平台的抓取。
  • 剩下的都是RSS的相关信息了，自己填写版权所有方，邮件地址等等，以上所有参数都会写入 RSS Feed 文件里
  1. 发布 Episode
  • Hexo 的所有博文都是 Markdown 语法写成的，不过对于播客来说套模板就好。
  • DDFPod 的 source/_post 目录里新建 <文件名>.md ，文件名会影响到网页链接，建议用英文，不带标点符号，可以用半角减号或下划线，文件内容如下：
  • 参数说明:
  • media 和 播客logo的配置类似，建议在 DDFPod 的 source 文件夹内新建 audio 文件夹，并放入 episode.mp3
  • image 也一样，不过是单集的 logo
  • length 填音频文件的大小，按字节计算，可以在文件右键属性里查看
  • mediatype 一般 MP3 用 audio/mpeg，WAV 用 audio/wav
  • duration 持续时间 小时:分钟:秒
  • chapter 和 CD 分轨类似，一般来说，只保留
  1. 生成和部署
  • 在 DDFPod 进入 Bash（在 DDFPod 文件夹上右键，选择 Git Bash Here）
  • 输入 hexo generate && hexo deploy 或者 hexo g && hexo d 并 Enter ，上传完成之后
    即可访问 https://用户名.github.io ，例：https://ddfcast.github.io
  • 第一次上传 Bash 会问你 Are you sure you want to continue connecting (yes/no)? 这时候敲 yes 然后 Enter 就好
  • 最后，GitHub有单个文件 100MB ，整个项目 10GB 的限制
